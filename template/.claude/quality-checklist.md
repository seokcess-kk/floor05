# 통합 품질 체크리스트

> 각 Phase가 끝날 때마다 해당 섹션을 확인하세요.
> 프로젝트 시작 전에는 Phase 0~5를 순서대로 점검합니다.

---

## Phase 0: Spec (기획서)

- [ ] 기술 스택이 정확한 버전까지 적혀있는가?
- [ ] 파일 경로가 복사-붙여넣기로 바로 쓸 수 있는 수준인가?
- [ ] 구현 계획이 한번에 하나씩 할 수 있는 작은 단위로 나뉘어 있는가?
- [ ] 완료 확인 방법이 실행 가능한 명령어로 되어 있는가?
- [ ] 성공 기준이 "됐는지 안 됐는지" 판단할 수 있게 구체적인가?

## Phase 1: CLAUDE.md + 기본 설정

- [ ] 프로젝트 폴더 구조가 트리 형태로 적혀있는가?
- [ ] 실행 명령어가 복사-붙여넣기로 바로 돌아가는가?
- [ ] 실수하기 쉬운 부분이 "절대 하지 말 것"으로 명시되어 있는가?
- [ ] 핵심 함수/클래스 이름이 정확하게 적혀있는가?
- [ ] 분량이 5~15KB 사이인가? (너무 짧으면 정보 부족, 너무 길면 토큰 낭비)
- [ ] "자동 리뷰 규칙" 섹션이 CLAUDE.md에 포함되어 있는가?
- [ ] .claude\settings.json에 Stop Hook이 설정되어 있는가?
- [ ] Stop Hook 명령어를 수동으로 한번 돌려서 정상 작동 확인했는가?

## Phase 2: Skills

- [ ] 각 SKILL.md가 100줄 이내인가?
- [ ] 상세 규칙이 resources/ 폴더에 분리되어 있는가?
- [ ] 스킬 description이 "언제 켜져야 하는지" 명확하게 적혀있는가?
- [ ] ✅ DO / ❌ DON'T에 구체적인 코드 예시가 있는가?

## Phase 3: skill-rules.json

- [ ] 모든 스킬에 keyword + intentPattern + fileTrigger가 있는가?
- [ ] 한국어와 영어 키워드가 모두 포함되어 있는가?
- [ ] pathPatterns가 실제 프로젝트 폴더 구조와 맞는가?
- [ ] 보안/브랜드/호환성처럼 반드시 지켜야 하는 것에 block 모드가 적용되어 있는가?
- [ ] 스킬 수가 10개 이하인가? (넘으면 토큰 낭비)

## Phase 4: Agent + 외부 기억 장치

- [ ] planner + plan-reviewer가 있는가?
- [ ] 각 에이전트에 성공 기준이 적혀있는가?
- [ ] 3대 문서(plan, context, tasks)가 작업 시작 전에 만들어졌는가?
- [ ] tasks.md의 "현재 상태" 섹션이 매 Phase마다 갱신되고 있는가?

## Phase 5: Hooks (추가 훅)

- [ ] Stop Hook은 Phase 1에서 이미 설정되어 정상 작동 중인가?
- [ ] 추가 Hook이 필요한 반복 문제가 발생했는가?
- [ ] 추가 Hook을 넣었다면 수동으로 테스트한 뒤 등록했는가?

---

## DDD (백엔드가 있는 경우)

- [ ] 첫 도메인의 3파일 세트(model/repository/service)를 직접 작성했는가?
- [ ] 비즈니스 용어가 그대로 코드명으로 쓰이고 있는가?
- [ ] 도메인 간에 직접 참조하지 않고 격리되어 있는가?
- [ ] 나머지 도메인은 "이 패턴 복제해서 만들어"로 지시했는가?

---

## 매 Phase 완료 시 공통 점검

- [ ] 빌드가 성공하는가?
- [ ] 린트 경고가 0개인가?
- [ ] 타입 체크가 통과하는가?
- [ ] tasks.md "현재 상태"가 갱신되었는가?
- [ ] 다음 Phase가 무엇인지 명확한가?

> 코드 품질(보안, 에러 처리, 타입 정확성, Spec 충족 여부)은
> code-architecture-reviewer 에이전트의 **셀프 체크 리마인더**가 점검합니다.
> 위 항목은 프로세스 관점, 셀프 체크는 코드 관점입니다.
