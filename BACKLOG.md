# floor05 기능 백로그 (경쟁사 리서치 기반)

> 최종 갱신: 2026-06-11
> 선도 서비스(iLoveIMG·TinyPNG·Squoosh·remove.bg·사람인/잡코리아·근로계산기·비볼디·알씨·picdefacer 등)를 리서치해 **흡수할 기능**을 정리한 실행 백로그. 1차 필터는 항상 **"서버 없이 브라우저에서 되는가"** — floor05의 $0·무전송·개인정보 차별점을 깨지 않는 선에서만 채택한다. [[GROWTH.md]]의 로드맵과 함께 본다.

## ⚠️ 먼저 짚을 두 가지 함정

1. **배경 제거 — 모델 라이선스 지뢰**
   웹에 흔한 누끼 데모(addyosmani/bg-remove 등)가 기본 쓰는 **RMBG-1.4 / RMBG-2.0 은 Bria 비상업 라이선스 → AdSense 수익 사이트에서 사용 시 위반**. `@imgly/background-removal` 은 **AGPL-3.0**(전체 소스 공개 의무) 또는 유료. → floor05는 **MODNet·ORMBG(Apache-2.0)** 또는 **BiRefNet/BiRefNet_lite(MIT)** 만 사용한다.
2. **연봉 계산기 — 정확성이 신뢰의 생명**
   경쟁사마다 요율·자녀세액공제액이 엇갈리고 출처 불명. floor05는 **국세청 간이세액표 원본을 JSON 상수로 임베드**, **4대보험 요율을 연도별 상수로 분리**, **"YYYY년 X월 기준" 명시**로 신뢰 도구로 포지셔닝한다. 근사식 사용 금지.

## 기존 6개 도구 품질 벤치마크 (Canvas 네이티브 vs 업계 코덱)

코드 확인 결과 floor05는 압축·리사이즈·변환·크롭·합치기를 **전부 순수 Canvas**(`toBlob`/`convertToBlob`, `drawImage`+`imageSmoothingQuality:'high'`)로 구현하고, MozJPEG/OxiPNG/Pica/AVIF 같은 WASM 코덱은 쓰지 않는다. 출시 운영엔 $0로 깔끔하지만, **출력 품질·효율**에서 다음 격차가 있다.

| 도구 | 현재 구현(코드 확인) | 업계 best | 현재 수준 | 핵심 격차 |
|---|---|---|---|---|
| **압축(JPEG)** | `toBlob('image/jpeg',q)`. 출력은 **항상 JPEG로 강제**(PNG/WebP 입력도 JPG로). 목표용량=품질 이진탐색→해상도 축소(견고) | MozJPEG | 보통 | 같은 체감 품질에서 **~15~20% 더 큰 파일**(추정, libjpeg-turbo 공식 평균 −20%) |
| **압축(PNG)** | 사실상 없음 — 압축 도구가 PNG를 **JPG로 바꿔버려 투명도 상실**. Canvas는 PNG 양자화·최적화 불가 | pngquant(−60~80%)+OxiPNG(−10~40%) | **미흡(최대 격차)** | "PNG 압축" 유입자가 투명 손실 + 진짜 PNG 압축 부재 |
| **변환(WebP)** | `toBlob('image/webp')` + **`isWebPSupported()`로 미지원 브라우저는 옵션 숨김**(올바른 방어) | libwebp(동일 계열) | 양호 | 코덱은 동일. 단 **FF/Safari 사용자는 WebP 출력 자체가 불가**(숨겨짐) |
| **변환(AVIF)** | **없음** | libavif(−59%급) | 미흡 | 차세대 포맷 출력 부재. Canvas 인코딩은 Chrome 124+만 |
| **리사이즈** | 단일 패스 `drawImage`+smoothing high | Pica(Lanczos)/다단계 축소 | 보통 | 큰 축소(4000→500 등)에서 흐림 가능. **다단계 축소는 순수 Canvas로도 $0 개선 가능** |
| **HEIC→JPG** | heic2any, **출력 JPG 고정·품질 0.92 하드코딩** | 다포맷 | 보통 | 출력 포맷/품질 선택 부재(Quick Win) |
| **크롭/합치기** | 기능 정상, Canvas 적합 | — | 양호 | 품질 격차 없음(기능 추가는 Tier 0/2) |

**이미 잘하고 있는 것(유지)**: 변환의 WebP 지원 가드, 압축 목표용량 이진탐색+해상도 폴백, Canvas 재인코딩에 따른 EXIF 자동 제거, OffscreenCanvas 워커+메인스레드 폴백. → 잘 설계돼 있고, 격차는 "인코더 효율" 한 축이다.

### Tier Q — 코덱 업그레이드 (jSquash, 전부 Apache-2.0/MIT · $0·무전송 유지)
WASM 코덱도 클라이언트 실행이라 서버비 0·파일 무전송 원칙 그대로. 비용은 번들 용량뿐 → **도구 진입 시 동적 로드**(heic2any 패턴 재사용). 격차 큰 순서로:
1. **PNG: `@jsquash/oxipng`(+imagequant) [최우선]** — 현재 최대 약점. "PNG 압축"을 진짜로 줄이거나(무손실 −10~40%, 손실 팔레트 −60~80%), 최소한 PNG 유지 압축 옵션 제공. *imagequant(libimagequant) 라이선스만 확인.*
2. **WebP/AVIF: `@jsquash/webp`,`@jsquash/avif` [고]** — FF/Safari에서도 WebP 출력 가능 + AVIF 차세대 포맷 추가(JPEG 대비 −59%급). 호환성 공백 해소.
3. **JPEG: `@jsquash/jpeg`(MozJPEG) [중]** — 동일 품질 ~15~20%↓. 단 인코딩 느림(약 50배) → 워커+진행률 UX 필수(이미 워커 구조 있음).
4. **리사이즈: 다단계 축소(무의존, 먼저) 또는 `pica`(MIT) [선택]** — 큰 축소 품질↑.
> ❌ 라이선스 주의: `wasm-vips`(LGPL, ~4.6MB)·imagequant(이중 라이선스)는 별도 검토. jSquash 모듈은 전부 Apache-2.0이라 안전.

## Tier 0 — Quick Win (기존 도구 보완, 거의 Canvas만으로 1~2일)

가장 싸고 빠른 ROI. 기존 도메인 권위를 그대로 타고, ML 불필요.

### 개인정보(전 도구 공통)
- [ ] **압축/변환 결과에 "메타데이터(EXIF·GPS) 제거됨" 명시** — 이미 Canvas 재인코딩으로 일어나는 일을 셀링포인트로 전면화. "파일 미전송" 메시지와 시너지. (근거: TinyPNG strip)

### 크롭
- [ ] **원형 크롭(circle crop)** — 프로필 사진 수요 큼. Canvas `clip(arc)` + 투명 PNG 출력
- [ ] **고정 크기 출력** — 비율뿐 아니라 "정확히 350×450px" 지정(크롭 후 리샘플)
- [ ] **증명사진 프리셋**(여권 3.5×4.5cm, 반명함 등 한국 규격) — 한국어 도구 빈틈
- [ ] 둥근 모서리(rounded corners) 옵션

### 리사이즈
- [ ] **"확대 안 함(do not enlarge)" 옵션** — 작은 이미지 강제 확대 방지 (iLoveIMG 표준)
- [ ] **여백/정사각 패딩** — 캔버스 확장 + 배경색/투명 + 3×3 앵커. 합치기의 배경 로직 재활용. 인스타 정방형·프로필 수요
- [ ] 25/50/75% 빠른 축소 버튼
- [ ] **여러 SNS 프리셋 동시 출력 → ZIP** (인스타+유튜브+카톡 한 번에). 조합 도구 부재 = 빈틈

### 변환 / HEIC
- [ ] PNG→JPG **배경색 커스텀**(로직상 `backgroundColor` 이미 받음, UI만 노출)
- [ ] 일괄 변환 시 **파일별 다른 포맷** 허용
- [ ] HEIC→**PNG/WebP 출력 선택**(heic2any `toType` 지원) + **품질 슬라이더 노출**(현재 0.92 고정)

### 합치기
- [ ] **그리드(N열) 배치** — 콜라주형. 리사이즈 여백 로직과 코드 공유

## Tier 1 — 신규 Room (저비용, 키워드 우주 확장)

### 텍스트 Room — 글자수 세기 (MVP)
100% 브라우저. 한국 사이트는 기능 단순 + 광고 도배 → 담백 UX + 한국 특화 디테일로 진입.
- [ ] 글자 수(공백 포함/제외, 실시간)
- [ ] **byte 수 — 2byte 기준 + 3byte 기준 둘 다**(공백 포함/제외). 자소서 byte 핵심
- [ ] 단어/줄/문장/문단 수
- [ ] **원고지 매수**(200자·400자 전환 + **남은 칸** 표시)
- [ ] **자소서 목표 프리셋**(500/1000/1500자) + 잔여 카운트, **공백 포함 기본값**
- [ ] **NAVER 기준 / HWP 기준 줄바꿈 토글**(저비용 차별점 — 카피킬러만 제공)
- [ ] 전체 복사 / 지우기, localStorage 자동저장(재방문 복원)
- [ ] FAQ(공백 포함?·byte 2/3?·원고지 환산?) + Schema
- 같은 Room 후속 도구: 공백/줄바꿈 정리, 중복 줄 제거·정렬, 대소문자 변환, 찾기·바꾸기, 한↔영타 변환
- ❌ 제외: 맞춤법/표절/AI 검사(서버 필요 → 원칙 위배)

### 계산기 Room — 연봉 실수령액 (MVP)
100% 브라우저 가능(요율=상수, 간이세액표=JSON 동적 로드). **정확성 최우선(위 함정 2 참조).**
- 입력: 연봉/월급 토글, 퇴직금 별도/포함, 부양가족 수, **8~20세 자녀 수(별도)**, 비과세 식대(기본 20만원, 2026 유지)
- 공제 6종: 국민연금 4.75%(상한 637만/하한 40만), 건강 3.595%, **장기요양=건보료×13.14%**, 고용 0.9%, 소득세(간이세액표 lookup), 지방세=소득세×10%
- 결과: 월 실수령액 + 항목별 표 + **실효세율** + (차별화) 월별 차트·시나리오 비교·URL 공유
- [ ] **연봉별 실수령액 정적 표/페이지**(`/tools/salary/[금액]`) — 롱테일 SEO 핵심(시그널플래너 모델)
- 동시 출시 묶음(우선순위): ①퇴직금 ②시급·주휴수당(+최저임금 2026=10,320원) ③4대보험료(로직 90% 공유) ④연봉별 정적표 ⑤(택1) 평수↔㎡ 또는 디데이
- 연도 버저닝: `constants/2026.ts` + `tax-table-2026.json`, 매년 1~3월 상수만 교체

## Tier 2 — 이미지 Room 심화 (신규 도구)

### 모자이크 / 블러 (개인정보 가리기)
한국 수요 큼(중고거래 번호판·명찰·주소). picdefacer가 브라우저 ML 가능함을 입증.
- 1단계(수동, Canvas만): **효과 선택(모자이크/가우시안 블러/블랙박스)**, 사각형 드래그 + 자유 브러시(크기 슬라이더), 강도 슬라이더, 다중 영역, Before/After, EXIF 제거
- 2단계(🧠 ML, 동적 로드): **face-api.js 자동 얼굴 감지 후 일괄 모자이크**(TensorFlow.js, 모델 수 MB) + 감도 프리셋. 번호판은 후순위
- 차별화: 한국어 + 100% 브라우저 + 무가입(알씨=설치형, iLoveIMG=서버 대비 빈틈)

### 워터마크 (SPEC §8.5 보강)
전부 Canvas, 브라우저 100%.
- 텍스트(글꼴/색/크기/두께, **그림자·외곽선**, 투명도) — **한글 폰트 포함이 차별화**
- 로고 PNG 업로드(크기·투명도)
- 배치: **9방향 위치 + 대각선 타일 반복(45°)+간격(span) 슬라이더**(iLoveIMG 미제공=빈틈) + 회전·여백
- **일괄 적용 + 프리셋 저장**
- 차별화: "워터마크 도구인데 **우리 워터마크는 안 붙임**"(Photoroom·Canva 무료판은 강제)

## Tier 3 — 배경 제거 / 누끼 (대형 트래픽, 빌드 中, 라이선스 주의)

브라우저 ML로 서버 없이 가능(transformers.js v3 + WebGPU/WASM, Xenova/modnet 공식 데모가 100% 클라이언트).
- **권고 스택**: **transformers.js + MODNet(Apache, fp16 13MB)** → 인물 중심 출시. heic2any처럼 **도구 진입 시 동적 로드** + WASM 폴백 + Web Worker
- 확장: 범용 객체는 **BiRefNet_lite(MIT, 339MB)** 를 "고품질(데스크톱/WebGPU) 모드"로 분리
- ❌ 금지: RMBG-1.4/2.0(Bria 비상업), @imgly(AGPL)
- 흡수 UX: 자동 1클릭 + **수동 보정 브러시(erase/restore)**, 투명 체커보드 미리보기, 단색/이미지 배경 교체
- 차별화 카피: **"사진을 서버에 안 올리는 누끼"** — 가입·워터마크·해상도 제한 없음
- 빌드 난점: 모델 다운로드 진행률 UX·캐싱(IndexedDB)·WebGPU 폴백 (ML 자체보다 이쪽)

## 추천 실행 순서

1. **Tier 0 Quick Win 묶음** — 기존 도구 보완(원형 크롭·증명사진 프리셋·정사각 패딩·메타데이터 제거 명시 등). 가장 싸고 즉시 SEO/UX 개선.
2. **글자수 세기**(텍스트 Room 개시) — 초저비용·초고수요.
3. **연봉 실수령액 + 퇴직금**(계산기 Room 개시) — 최고 CPC. 정확성 검증에 시간 투자.
4. **모자이크/블러** 또는 **워터마크**(이미지 Room 심화).
5. **배경 제거** — 공들여 준비(라이선스·모델 로딩 UX). 트래픽 폭발력 최대.

> 전 항목 공통: SEO title `[키워드] - [차별화 한 마디]`(브랜드명 제외), FAQ+Schema, 도구 본체 밖 광고 최대 3개, 담백 보이스 유지.
